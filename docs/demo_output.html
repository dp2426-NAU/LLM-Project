ABSTRACT LoRA (Low-Rank Adaptation) and full fine-tuning represent two contrasting strategies for adapting pretrained large language models to downstream tasks. Full fine-tuning updates all model parameters, offering maximal expressiveness at the cost of computational overhead and storage requirements proportional to model size. LoRA constrains updates to low-rank decomposition matrices injected into specific weight layers, reducing trainable parameters by orders of magnitude while preserving competitive task performance. [source: transformers_overview, chunk_3] MAIN FINDINGS 1. Parameter Efficiency: LoRA reduces trainable parameter count by approximately 10,000× relative to full fine-tuning for models in the 7B–70B parameter range. A GPT-3-scale model fine-tuned with LoRA (rank=4) requires ~4.7M trainable parameters vs. 175B for full fine-tuning. [source: llm_finetuning, chunk_1] 2. Task Performance: Across standard NLP benchmarks (GLUE, SuperGLUE, MMLU), LoRA matches or exceeds full fine-tuning performance within 1–3% on most tasks. Performance gaps widen on tasks requiring broad distributional shift from the base model's pretraining distribution. [source: llm_finetuning, chunk_2] 3. Computational Cost: Full fine-tuning of a 7B model requires ~112 GB GPU memory (bf16 weights + gradients + optimizer states). LoRA with the same model requires ~18 GB, enabling fine-tuning on a single A100 or consumer-grade GPU cluster. [source: transformers_overview, chunk_7] 4. Serving Overhead: LoRA adapters add negligible inference latency when merged into base weights post-training. Multiple adapters can be maintained for different tasks and swapped at inference time, enabling multi-task serving from a single base model. [source: llm_finetuning, chunk_5] CRITICAL ANALYSIS Two evidence gaps were identified in the retrieved corpus: - No empirical comparison of LoRA vs. full fine-tuning on code generation tasks (HumanEval, MBPP) - Training stability characteristics under limited-data regimes (<1K examples) are underrepresented SYNTHESIS & RECOMMENDATION For practitioners adapting LLMs to domain-specific tasks, LoRA is the recommended default: it delivers comparable accuracy at <5% of the compute cost, is compatible with quantized base models (QLoRA), and supports rapid iteration through adapter swapping. Full fine-tuning is warranted only when the target distribution diverges substantially from pretraining data or when maximum expressiveness is required for safety-critical applications.
